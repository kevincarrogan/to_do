BackboneJS

What's that?

BackboneJS is a javascript framework based on the Model-View-Presenter (MVP) application design paradigm.

What does BackboneJS provide?

  • Backbone.Model: Manage datas
  • Backbone.Collection: Set of models
  • Backbone.View: Organize the interface
  • Backbone.Events: Extra module to customize events
  • Backbone.Route: Define relations URL - view

To-Do List!

To-Do List is a single-page web application where you can create/edit/remove tasks.

It has been made just as a example of this workshop, using Django on backend side and BackboneJS to build the frontend site. Others requirements are: Postgres, UnderscoreJS, requireJS and jQuery.

The data model of this example is not hard at all!

  • TaskModel: It is a Backbone.Model that store all datas related with one task.
  • TasksCollection: It is a Backbone.Collection that is used as a set of all tasks in the list.
  • TaskView: It is a Backbone.View that manage the DOM of just one task.
  • TasksView: It is a Backbone.View that handle the basic funcitonality we can do with a task: Add / Edit / Remove / Restore.
  • BaseNavView: It is a Backbone.View that handle all customs events that could be triggered for each different navigation bars. Based on this view, it defines: AddTaskNav, CreateFormNav, EditRemoveNav, SelectStatusNav and SaveCancelNav.
  • NavView: It is a Backbone.View that handle five different navigation bars.
  • Broker: It is a Backbone.Events object to bind customs events on it.

Requeriment: UnderscoreJS

UnderscoreJS is the only hard dependency that BackboneJS has. It provides a set of methods on Collections (such as .each()) and it brings much more of them to use in views. Really useful.

Some of them are:

  • _.each(list, iterator): Iterates over a list of elements. Function gets '(element, index, list)' as arguments. If 'list' is actually a Javascript Object, arguments will be '(key, value, list)'
  • _filter(list, iterator): Looks throught each value in the list and return an array withh all the values that pass a truth test (iterator).
  • _.bindAll(object, *methodNames): Binds a number of methods on the object, specified by methodNames, to be run in the context of that object whenever they are invoked.
  • _.extend(object, *sources): Copy all of the properties in the source objects over to the destination object, and return the destination object.
  • _.template(templateString, [datas]): Compiles JavaScript templates into functions that can be evaluated for rendering using JSON data source.

Views: TasksView (5)

        selectItems: function (model) {

          // This will toggle between bars. When there is no task selected the only action the user
          // has is 'create a task'. If one or more tasks are selected, the actions are 'edit tasks',
          // 'remove tasks' and 'change status'.

          if (model.get('selected') && tasks_collection.where({selected:true}).length === 1) {
            // model has been the first task to be selected in the list of tasks.

            this.broker.trigger('task:select', 'checked');

          } else if (!model.get('selected') && !tasks_collection.wherer({selected:false}).length) {
            // model has been the last task selected in the list and it becames unselect.

            this.broker.trigger('task:select', 'unchecked');
          }
        },

Views: TasksView (7)

        addTask: function (object) {
          var self = this, view;
          // This function does not get any model yet. What it has is an Javascript object as JSON
          // with all the datas needed to create a new task. 
          this.tasksCollection.create(object, {
            success: function (model) {
              // Creating new TaskView with the model just created.
              view = new TaskView({
                model: model,
                className: model.get('status')
              });
              // Add the view into the private collection.
              self._viewCollection.add({'id': model.get('id'), 'view': view});

              // If there was no task before, remove the 'non-task' item from the list.
              if (self.$el.find('.no-task').length) { self.$el.children().remove(); }
              self.$el.prepend(view.render().el);
            }
          });
        },

        editTasks: function () {
          var self = this, view;

          // When this function has been called, every selected task should be rendered as 'editMode'.
          // It means, it will be render using input:text rather than label on title to be edited.

          _.each(this.tasksCollection.where({selected:true}), function (model) {

            // It needs to get the view related with the model that is selected (model).
            view = self._viewsCollection.findWhere({'id': model.get('id')}).get('view');

            // It needs to be render as form view. For that, TaskView.render() has been modified.
            view.render('form');
          
          });

          this.editingFormEnable = true;
        },

Views: TasksView (10)

        saveTasks: function (action) {
          var self = this, view, title;
          _.each(this.tasksCollection.where({selected:true}), function (model) {
            view = self._viewsCollection.findWhere({'id': model.get('id')}).get('view');
            
            // This function could be called by saving a new title or changing status.
            if (action === 'saving') {
              title = view.$el.find('[name=task_title_' + model.get('id') + ']').val();
              if (title) view.model.set('title', title);
              else if (view.model.get('new_status')) { 
                view.model.set('status', view.model.get('new_status')); }
              view.model.save();
            }

            view.render(); // Re-render taskView updating the DOM for each task.
          });

          this.editingFormEnable = false;
        },

Views: TasksView (12)

        removeTasks: function (action) {
          var self = this, view;

          if (action === 'saving') {
            _.each(this.tasksCollection.where({selected:true}), function (model) {
              view = self._viewsCollection.findWhere({'id': model.get('id')}).get('view');
              view.remove();
              self._viewsCollection.remove(self._viewsCollection.findWhere({'id': model.get('id')}));
            });

            this.tasksCollection.remove(this.tasksCollection.where({selected:true}));

            // If the user remove all task in the list it needs to render the 'no-task' item.
            if (!this.tasksCollection.length) {
              this.render();
            }
          }

          this.editingFormEnable = false;
        },
